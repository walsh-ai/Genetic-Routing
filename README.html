Files

  • README.txt Short README document that can be considred instead of this hyperlinks page
  • Technical Report The final technical report provided for this project; note that this report includes four appendices that are not counted towards the word limit.
  • Barabasi-Albert JN Jupyter Notebok in which the Barabasi random graph model is implemented and the properties of the generated graphs verified.
  • Dynamic BGA JN Jupyter notebook simulations in which the Best of Generation Accuracy of the GA variations for dynamic topologies is calculated.
  • Dynamic BGA 0.025 JN Jupyter Notebook in which the BGA simulations are run again with mutation rate of 0.025 as opposed to 0.03 used originally in Dynamic BGA JN
  • Dynamic Series JNSimulations of each GA variation for each topology series including dynamics.
  • Dynamic Auto JN An unfinished set of simulations in which the BGF for each topology series would be calculated with all GA variations where the GA class would be modified to automatically adjust the mutation rate and population size in response to the size/type of graph based on the optimal values ascertained in previous simulations (E.g. WS 100 has optimum population size 75)
  • Dynamic Probability JN Jupyter notebook simulations: How does the change probability effect GA performance (HIGA) for different dynamic models?
  • Edge Set Dynamics JN Jupyter notebook implementing a function to update the edge set of a Networkx graph randomly; used in dynamic simulations.
  • Weight Set Dynamics JN Jupyter notebook implementing a function to update the edge weights of a Networkx graph randomly; used in dynamic simulations
  • EIGA JN Jupyter notebook implementing the Elitism Immigrant GA variation.
  • Erdos-Reynyi JN Jupyter notebook implementing the Erdos-Renyi random graph model
  • Optimum Population JN Jupyter notebook of simulations used to determine the optimum population size, where generally population size of 100 performs best except for WS graphs where ~75 shows good results
  • HIGA Pop Test to determine whether HIGA benefits from a larger population size than the 100 hypotheses optimum for the Specialised GA without immigrants.
  • Mutation Rate Simulations over static graphs for the Specialised GA without immigrant schemes, to identify the optimum mutation rate for the GA.
  • GA Class Python class implementing the GA with immigrant schemes for DSPRP simulations, giving methods for initialising the population over a new graph, excuting a retrieving results from a single generation at a time and other useful methods for simulations.
  • HIGA Implementation of Specialised GA for SSSP with Hybrid Random-Elitism immigrant scheme.
  • Log-normal Method of generating random graphs with Log-Normal degree distribution is implemented and the statistical properties of generated graphs are verified
  • Mutation Test Exploration of a proposed alternative mutation method designed to solve performance issues that were ultimately attributed to the initial population generation.
  • Node Set Dynamics Jupyter notebook implementing the node set dynamic model used in simulations, and simulations of the effect of the Node Set dynamics on the shortest paths and path lengths in networks.
  • WS JN Jupyter notebook implementing the Watts-Strogatz model and investigating the statistical properties of the graphs generated with this model. Graphs are given as Networkx objects.
  • Proto_ga TSP Prototypical GA proposed by Tom Mitchell adapted for the TSP;
  • Proto-ga TSP #2 Prototypical GA proposed by Tom Mitchell adapted for the travelling salesman problem.
  • RIGA Implementation of the Specialised GA with Random Immigrants Scheme.
  • Runtime Jupyter notebook examining the runtime of the Specialised GA divided by time to generate the initial random population and to converge the GA. Population generation is the most expensive operation which scales poorly with the size of the network.
  • Specialised GA #1 The specialised GA for the SSSP problem.
  • Specialised GA #2 This variation of the original Specialised GA for the SSSP problem utilises and tests and different mutation method. Performance issues for large graphs were hypothesised to have been caused by the expensive mutation method. This alternative offered worse fitness and similar computation time and hence it was not adopted.
  • Static Performance Simulations testing the BGF of GA variations for different static topologies. Elitism strategies HIGA, EIGA were found to perform well overall which may be biased towards the static topologies where less diversity is required.
  • TSP Bad GA The first GA programmed in this project, for the Travelling Salesman Problem (TSP) which performed poorly and was ill-conceived.
